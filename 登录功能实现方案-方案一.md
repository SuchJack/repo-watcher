# 登录功能实现方案（方案一：单用户 + 固定密码）

本文档描述如何为「仓库变更监控工具」实现**登录后才能进入后台**的能力，采用**单用户 + 固定密码**方式，避免未授权用户修改配置与仓库数据。

---

## 一、方案概述

- **适用场景**：个人或小团队自用、Docker 部署、无需多用户与数据库。
- **认证方式**：后端配置一个管理员密码（或用户名+密码），登录校验通过后颁发 **Session** 或 **JWT**，后续请求携带该凭证。
- **保护范围**：所有后台页面（仪表盘、仓库管理、系统设置）及所有写操作与敏感读操作的 API，未登录一律拒绝。

---

## 二、后端实现要点

### 2.1 密码存储

- 在 `backend/config.py` 的 `DEFAULT_CONFIG` 中增加字段，例如：
  - `admin_password`：管理员密码的 **bcrypt 哈希**（不要存明文）。
- 或通过**环境变量**传入，如 `ADMIN_PASSWORD`，便于 Docker 部署时配置。
- 校验时使用 `bcrypt.checkpw(plain_password.encode(), stored_hash.encode())`。

### 2.2 登录接口

- **路由**：`POST /api/auth/login`
- **请求体**：`{ "username": "admin", "password": "xxx" }`
- **逻辑**：校验用户名与密码；通过则：
  - **Session 方式**：在服务端存储 session（内存或 Redis），Set-Cookie（HttpOnly），返回 `{ "ok": true }`；
  - **JWT 方式**：签发 access_token（建议过期时间如 24 小时），返回 `{ "ok": true, "token": "..." }`。
- **失败**：返回 401 及错误信息。

### 2.3 鉴权方式

| 方式   | 实现要点 |
|--------|----------|
| Session | 使用 FastAPI 的 SessionMiddleware，登录时写 session，后续接口通过依赖项读取 session，未登录则 `HTTPException(401)`。 |
| JWT     | 使用 FastAPI 的 `Depends()`，从请求头 `Authorization: Bearer <token>` 解析 JWT，验证签名与过期时间，无效则 401。 |

### 2.4 需要保护的 API

以下接口必须要求**已登录**（通过 Session 或 JWT 依赖项校验）：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET  | `/api/repos`           | 仓库列表（若希望完全后台化则保护） |
| POST | `/api/repos`           | 新增仓库 |
| PUT  | `/api/repos/{id}`      | 修改仓库 |
| DELETE | `/api/repos/{id}`    | 删除仓库 |
| POST | `/api/repos/{id}/clear-updates` | 清空该仓库更新次数 |
| GET  | `/api/config`         | 获取配置 |
| PUT  | `/api/config`         | 保存配置 |
| POST | `/api/check`          | 立即检查 |
| GET  | `/api/state`          | 获取状态 |
| POST | `/api/state/clear-all-updates` | 清空所有更新次数 |

可选：`GET /api/auth/me` 用于前端判断当前是否已登录；`POST /api/auth/logout` 用于登出（Session 时销毁 session）。

## 三、前端实现要点

### 3.1 登录页

- **路由**：`/login`
- **组件**：新建 `frontend/src/views/Login.vue`
  - 表单：用户名（可固定为 `admin`）、密码
  - 提交调用 `POST /api/auth/login`，成功则保存凭证并跳转到 `/`
  - 使用 Element Plus 的 `el-form`、`el-input`、`el-button` 等

### 3.2 路由守卫

- 在 `frontend/src/router/index.js` 中配置 `router.beforeEach`：
  - 访问 `/login` 且已登录 → 重定向到 `/`
  - 访问 `/`、`/repos`、`/settings` 且**未登录** → 重定向到 `/login`
- 「已登录」判断：Session 方式看 Cookie；JWT 方式看 localStorage/内存中是否有有效 token（可选：请求 `GET /api/auth/me` 校验）。

### 3.3 请求携带凭证

- **Session**：在 `frontend/src/api/request.js` 中为 axios 设置 `withCredentials: true`，保证 Cookie 随请求发送。
- **JWT**：在 axios 请求拦截器中从 localStorage 读取 token，设置 `Authorization: Bearer <token>`；响应拦截器中若收到 401，则清除 token 并跳转 `/login`。

### 3.4 登出

- 前端清除 Cookie（Session）或删除 JWT。
- 若有 `POST /api/auth/logout`，可先调用再跳转 `/login`。
- 在 `App.vue` 头部导航中增加「退出」按钮，点击后执行登出并跳转登录页。

---

## 四、安全注意事项

1. **HTTPS**：生产环境务必使用 HTTPS，避免密码与 token 在传输中被窃听。
2. **密码存储**：仅存储 bcrypt 哈希，不存明文。
3. **JWT**：不在 payload 中存放敏感信息；设置合理过期时间（如 24 小时）。
4. **CORS**：若前后端分离部署，需正确配置 `allow_credentials` 与 `allow_origins`，避免 Cookie 无法携带。

---

## 五、实现顺序建议

1. 后端：增加密码配置/环境变量、bcrypt 校验、`POST /api/auth/login`（及可选 `GET /api/auth/me`、`POST /api/auth/logout`）。
2. 后端：实现鉴权依赖（Session 或 JWT），并在上述需要保护的接口上挂载该依赖。
3. 前端：新建登录页 `Login.vue`，路由 `/login`，表单调用登录接口并保存凭证。
4. 前端：在 `router/index.js` 中添加 `beforeEach` 路由守卫。
5. 前端：在 `request.js` 中配置携带凭证（Cookie 或 Authorization 头）及 401 时跳转登录。
6. 前端：在 `App.vue` 中增加「退出」按钮并实现登出逻辑。

按以上顺序实现后，只有登录成功的用户才能进入后台并调用受保护的 API，从而避免其他用户乱改配置与仓库数据。

## 六、默认密码与首次配置

**默认初始密码**：未配置任何管理员密码时，可使用用户名 **admin**、密码 **admin** 登录（仅当 config 中无 `admin_password` 且未设置环境变量 `ADMIN_PASSWORD` 时生效）。**首次登录后请在「系统设置 → 修改密码」中修改密码。**

自定义密码仍支持两种方式任选其一：

1. **环境变量（推荐用于 Docker / 首次启动）**  
   启动后端前设置环境变量 `ADMIN_PASSWORD` 为明文密码，例如：
   - Linux/macOS：`export ADMIN_PASSWORD=你的密码`
   - Docker Compose：在 `environment` 中增加 `ADMIN_PASSWORD: "你的密码"`

2. **写入 config.json（持久化）**  
   在 `backend/data/config.json` 中增加字段 `"admin_password": "<bcrypt 哈希>"`。  
   生成哈希示例（在项目根目录执行）：
   ```bash
   python -c "import bcrypt; print(bcrypt.hashpw(b'你的密码', bcrypt.gensalt()).decode())"
   ```
   将输出字符串填入 config 的 `admin_password`。若同时设置了环境变量，优先使用 config 中的哈希。

**注意**：若同时设置了环境变量或 config 中已有 `admin_password`，则不再使用默认密码（admin）。生产环境建议设置 `JWT_SECRET` 环境变量以更换默认 JWT 密钥。

---

## 七、修改密码

- **接口**：`PUT /api/auth/password`（需已登录）
- **请求体**：`{ "old_password": "当前密码", "new_password": "新密码" }`
- **前端**：在「系统设置」页的「修改密码」标签中填写当前密码、新密码、确认新密码，提交后密码写入 config，并提示重新登录（前端会清除 token 并跳转登录页）。
- **校验**：新密码至少 6 位；当前密码错误时返回 400。
